# プロダクトスコープ

## 一言で

**リポジトリやディレクトリ構成を、読み取り専用で美しく見せる React 用 UI。**  
ASCII の箱罫ではなく、左側の接続線と等幅フォントの行で階層を表す。

## 想定している利用シーン

- ブログ・ドキュメントの「プロジェクト構成」ブロック
- OSS の README やランディングでのファイルツリー風プレビュー
- 技術記事内のサンプルリポジトリ構成（`RepoFolderTree` など）

## 設計上の優先順位

1. **依存を薄く** — React を peer にし、他 UI ライブラリに引っかからない
2. **スタイルはプレーン CSS** — Tailwind 不要。`--ft-*` でテーマ可能
3. **パッケージ品質** — ESM/CJS デュアル、`exports`、型、publint / attw で検証
4. **読み取り専用が既定** — インタラクションは別レイヤーで検討（スコープ外を明確に）

## スコープ外（現時点で本パッケージの主目的ではない）

次のような要件は、**別コンポーネント・別パッケージ・利用側の実装**で扱う前提です。

| 領域 | 理由の概要 |
|------|------------|
| 開閉可能な `tree` ロールとキーボードフォーカス管理 | WAI-ARIA Tree Pattern の実装コストと API 表面積が大きい |
| ドラッグ＆ドロップ、名前変更、CRUD | エディタ／ファイルマネージャー向けの別プロダクトに近い |
| 仮想スクロール・数万ノード最適化 | データ可視化用途は要件が異なる |
| ファイルシステムや Git への直接バインド | データは `FolderTreeNode` として利用側が渡す |

「将来入れるかもしれない」アイデアは [backlog.md](./backlog.md) に置き、**製品の約束（README に書くこと）**と切り離して管理します。
